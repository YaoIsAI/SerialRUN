Quick Start

  1. Connect your serial device to the computer via USB
  2. Click the refresh button ↻ in the left panel to detect ports
  3. Select a serial port from the dropdown (e.g., COM3)
  4. Set the baud rate (default 115200, supports custom values)
  5. Click the green Connect button
  6. Type commands in the bottom terminal input box and click Send
Tip: If you don't know the baud rate, click the Auto button to auto-detect (progress shown in status bar).

Terminal Features

The terminal displays all send/receive data with color-coded directions:

  • Green ↓ RX — Received data
  • Blue ↑ TX — Sent data
  • Yellow ⚙ SYS — System messages

Toolbar Controls

  • Timestamp — Show time before each message
  • Auto Scroll — Auto-scroll to latest message
  • CRC — Auto-append checksum on send
  • DTR/RTS — Control serial signals (when connected)
  • Auto Send — Repeat send at set interval
  • T/O — RX timeout (auto or manual)
Auto-Wrapping: The toolbar uses adaptive layout — controls automatically wrap when the window is narrow.

HEX Mode

Switch between TXT and HEX mode in the input box. HEX mode auto-strips spaces and 0x prefixes.

RX Timeout (T/O)

Controls how long to wait for more data after initial read:

  • Auto — Calculated by baud rate (check T/O checkbox)
  • Manual — Set custom value 10-2000ms (uncheck T/O)

Right-Click Menu

  • Copy — Copy current line
  • Copy All — Copy all terminal data
  • Convert to HEX — Show text as hex
  • Resend — Re-send the line data (TX only)

Modbus Debug

Quick register read/write with 8 function codes and configurable response timeout.

Quick Request

  • Set Slave ID (0-247), function code, start address
  • For reads: set quantity; for writes: set value
  • Response Timeout — 50-5000ms, default 500ms
  • TX/RX data also shown in terminal panel

Register Monitor

Set poll interval (100-10000ms) and view register values in real-time with address, raw value, and formatted display.

PLC Control

Professional PLC monitoring with brand presets and batch register operations.

  • Brands: Siemens S7-1200, Mitsubishi FX3U, Delta DVP, Omron CP1H, Custom
  • Data Types: BOOL, UINT16, INT16, UINT32, FLOAT32
  • Batch Read: Contiguous registers merged into single Modbus request
  • Inline Write: Click register row to edit and write values
  • Status Dots: Green/yellow/red = fresh (<3s) / aging (<10s) / stale

TCP/RTU Bridge

Bridge Modbus TCP clients to serial RTU devices. External SCADA/HMI software can communicate with serial devices via TCP.

  • Set TCP listen port (default 502)
  • Select serial port and baud rate
  • Set response timeout
  • Click Start to activate the bridge

MCP Server (AI Integration)

Built-in MCP server allows AI assistants to remotely control serial devices via TCP.

Setup

  1. In left panel, find MCP Server
  2. Check Enable MCP Server
  3. Set port (default 9527) and bind address
  4. Click Copy MCP Info to share with AI assistant

Plugins

SerialRUN supports plugins to extend its functionality. Plugins are Rust-based dynamic libraries that can add custom protocol support, UI panels, and more.

Installing Plugins

  • Open the Plugins panel from the top toolbar
  • Click Import ZIP and select a plugin.zip file
  • The plugin will be discovered and listed in the Installed tab
  • Enable/disable plugins using the toggle switch on each plugin card

Developing Plugins

Interested in building your own plugins? Visit the Plugin Development page for a complete guide with code examples, API reference, and example plugins.

Plugin Community

Local plugin import is fully functional. Community plugin marketplace is coming in a future release. Developers are welcome to build and share plugins now!

Tips & Troubleshooting

Tips

  • Start with 115200 baud rate for new devices
  • Use Auto button if you don't know the baud rate
  • HEX mode auto-strips spaces and 0x prefixes
  • Append CRC checksums for data integrity
  • Use Record/Replay for repeated test sequences
  • All data auto-saves to ~/.serialrun/ directory

FAQ

QuestionAnswer
Port list empty?Click refresh. Check device connection and drivers. Linux: sudo usermod -a -G dialout $USER
No data received?Check baud rate matches device, verify TX/RX wiring, check flow control.
Auto baud detection fails?Ensure device is sending data during detection.
Modbus no response?Check slave address, function code, address range, and CRC.
File transfer fails?Ensure both sides use same protocol, try lower baud rate.
